# Contributing(贡献指南)

## Change Types(变更类型)

- **ADR**:用于具有长期影响的架构决策
- **RFC**:用于在采纳之前需要讨论的提议变更
- **Standard update(规范更新)**:用于对现有工程规范的修订

## Flow(流程)

1. 记录问题陈述与范围。
2. 如果变更会影响共享行为,以 ADR 或 RFC 的形式记录所提议的变更。
3. 识别受影响的规范、仓库与团队。
4. 就安全、运维风险与实现契合度进行评审。
5. 在更新规范或相关指导之前先获得批准。

## Pull Request Requirements(Pull Request 要求)

- 清晰地总结变更了什么以及为什么
- 为变更提供证据或参考资料
- 明确列出假设与未决问题
- 针对安全与运维影响的评审说明
- 不含密钥、state 文件,或环境专属的凭据

## Standard Change Approval(规范变更审批)

- 小的编辑性修订可由仓库维护者批准
- 会改变行为的更新需要同行评审(peer review)
- 安全、存储、备份与平台方面的变更需要相应的领域评审者
- 会影响生产实践的变更在采纳前需要有书面记录的批准
