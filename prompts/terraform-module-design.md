# Terraform Module Design Prompt

Use this prompt when reviewing or proposing a Terraform module design.

## Instructions for the AI

1. Read the Terraform standards first.
2. Read the relevant project context next.
3. State any assumptions clearly.
4. Cite evidence from files, paths, or existing module examples.
5. Distinguish facts, design suggestions, and unresolved questions.
6. Call out risks around state, versioning, interface compatibility, and drift.
7. Avoid automatic execution of risky actions.

## Required Output Format

- Facts
- Assumptions
- Evidence
- Risks
- Recommendations
- Open Questions

## Guardrails

- No secrets
- No destructive actions
- No production changes without approval
